# Running CSI-NN2 Models on Bare-Metal C906 RTL Simulation

This document describes the end-to-end procedure for compiling a CSI-NN2
neural network model (`conv_softmax/model.c`) and running it on the OpenC906
bare-metal RTL simulation platform using VCS.

## 1. Overview

The goal is to run an HHB-generated Conv2D + Softmax INT8 quantized model on
the C906 processor core in RTL simulation.  The model was generated by the
T-Head HHB (Heterogeneous Honey Badger) toolchain and uses the CSI-NN2 API for
graph construction and inference.

**Model architecture:**
- Input: `int8 [1, 16, 28, 28]` (NCHW)
- Conv2D: 16→2 channels, 3×3 kernel, stride 1, pad 1
- Softmax: axis 1
- Output: `int8 [1, 2, 28, 28]`

**Key challenge:** The CSI-NN2 library is designed for Linux/RTOS targets with
full libc, but the OpenC906 testbench is a minimal bare-metal environment with
no OS, no dynamic linker, and limited memory.  Several RTL-level bugs surfaced
during bring-up that caused simulation hangs (timeouts).

## 2. Prerequisites

| Component | Path / Version |
|-----------|---------------|
| Toolchain | `Xuantie-900-gcc-elf-newlib-x86_64-V3.3.0` (GCC 14.3.0) |
| CMake | ≥ 3.3 (tested with 3.28.4) |
| VCS | Synopsys VCS (tested with X-2025.06-SP1) |
| CSI-NN2 | Git submodule at `csi-nn2/` |

The toolchain **must** be the `elf-newlib` variant (not `linux-gnu`), since we
target bare-metal with newlib as the C library.

> **Note:** GCC 14.3 supports RVV 1.0 intrinsics only.  The C906 implements
> RVV 0.7.1.  Therefore we cannot use the C906/RVV-optimized kernels and must
> fall back to the **reference C implementation**.

## 3. Step 1 — Build the CSI-NN2 Static Library

### 3.1 CMake Configuration

The library is cross-compiled for bare-metal RISC-V using
`-DCMAKE_SYSTEM_NAME=Generic` (tells CMake not to look for a host OS):

```bash
cd csi-nn2
mkdir c906_elf_build && cd c906_elf_build

cmake .. \
  -DCMAKE_SYSTEM_NAME=Generic \
  -DCMAKE_C_COMPILER=riscv64-unknown-elf-gcc \
  -DCMAKE_ASM_COMPILER=riscv64-unknown-elf-gcc \
  -DCONFIG_BUILD_RISCV_ELF_C906=ON \
  -DCMAKE_INSTALL_PREFIX=../install_nn2/c906

make -j8
make install
```

### 3.2 Key Build Flags

The `CMakeLists.txt` was modified to produce a bare-metal-friendly library:

```cmake
set(C906_BUILD_FLAGS
    -Os                           # Size optimization
    -ffunction-sections           # Enable --gc-sections at link time
    -fdata-sections
    -fno-optimize-sibling-calls   # ★ CRITICAL: prevents tail-call optimization
    -ffp-contract=off
    -march=rv64imafdc_zfh_zifencei_xtheadc
    -mabi=lp64d
    -mcmodel=medany
    -DSHL_BUILD_C906 -DSHL_BUILD_REF -DSHL_BUILD_GREF -DSHL_BUILD_RTOS)
```

### 3.3 What Was Disabled

| Feature | Why Disabled |
|---------|-------------|
| `SHL_DEBUG` | Calls `printf`/`vfprintf` which may attempt file I/O |
| `SHL_TRACE` | Allocates trace event buffers, uses file I/O |
| `SHL_LAYER_BENCHMARK` | Uses `clock_gettime` (unavailable bare-metal) |
| RVV / C906 optimized sources | Require RVV 0.7.1 intrinsics (incompatible with GCC 14.3) |

### 3.4 Source Selection

Only three source groups are included (no vector-optimized code):

```cmake
LIST(APPEND C906_LST ${NN2_SRCS} ${REF_SRCS} ${GREF_SRCS})
```

- `NN2_SRCS` — Core API, session management, utilities
- `REF_SRCS` — Reference C implementations of all operators
- `GREF_SRCS` — Graph-reference backend (builds compute graph, dispatches to reference ops)

### 3.5 Stub Header

`shl_profiler.h` includes `<sys/syscall.h>` which doesn't exist in newlib.
A minimal stub is provided at `smart_run/tests/cases/conv_softmax/stubs/sys/syscall.h`
and added to the include path via `target_include_directories()`.

**Output:** `csi-nn2/install_nn2/c906/lib/libshl_c906_rtos.a` (~2.2 MB)

## 4. Step 2 — Create Bare-Metal Test Infrastructure

Four files were created in `smart_run/tests/cases/conv_softmax/`:

### 4.1 `model.c` — The Neural Network Model

This is the HHB-generated model file.  One change was required:

```c
// Original: CSINN_C906 (requires C906-specific op registrations we can't compile)
// Changed to: CSINN_REF (uses reference + graph-reference backend)
sess->base_api = CSINN_REF;
```

The run mode remains `CSINN_RM_CPU_GRAPH` (graph mode), which builds a compute
graph first, then executes it during inference.

### 4.2 `bare_main.c` — Entry Point

```c
int main(void) {
    install_trap_handler();           // Override crt0.s broken vector table
    void *sess = csinn_((char *)model_params);  // Build graph
    // ... set up input tensor with dummy data ...
    csinn_update_input_and_run(inputs, sess);    // Run inference
    // ... cleanup ...
    return 0;
}
```

A custom **trap handler** is installed that redirects any exception to the
`__fail` label (which the testbench detects as test failure).  This is necessary
because the default `crt0.s` exception vector table uses `.long` (4-byte)
entries but the handler loads them with `ld` (8-byte), causing an infinite
exception loop on any trap.

### 4.3 `test_data.h` — Embedded Weights and Input

```c
static const unsigned char model_params[496] = { ... };  // Quantization params + weights
static const signed char input_data[12544] = {0};        // Dummy zeros: [1,16,28,28]
```

Weights are embedded as a C array (extracted from the HHB model binary).
Input data is all zeros — sufficient to verify the computation completes.

### 4.4 `sbrk.c` — Heap and Stubs

Provides three critical pieces:

1. **`_sbrk()`** — Heap allocator for newlib's `malloc`.  Grows from the
   linker-defined `end` symbol toward the stack.

2. **`shl_target_init_c906()`** — Empty stub replacing the real C906 backend
   init (which would register RVV-optimized ops we can't compile).

3. **`shl_get_runtime_callback()` override** — Bypasses a critical RTL bug
   (see [Section 7](#7-debugging-simulation-timeouts--the-hard-part)).

4. **`shl_trace_move_events()` stub** — Called directly (not through
   `SHL_TRACE_CALL` macro) from `graph_ref/setup.c`, so needs a stub when
   `SHL_TRACE` is disabled.

## 5. Step 3 — Integrate into the Build System

### 5.1 `smart_run/tests/lib/Makefile`

Added `EXTRA_CFLAGS` and `EXTRA_LDFLAGS` variables so the `conv_softmax` recipe
can pass CSI-NN2 include/library paths without modifying the shared Makefile:

```makefile
CFLAGS += ... ${EXTRA_CFLAGS}
# Link line:
${EXTRA_LDFLAGS} -lm
```

### 5.2 `smart_run/setup/smart_cfg.mk`

Added `conv_softmax` to `CASE_LIST` and a build recipe:

```makefile
CSI_NN2_INSTALL := ../../csi-nn2/install_nn2/c906
conv_softmax_build:
    @cp ./tests/cases/conv_softmax/bare_main.c ./work
    @cp ./tests/cases/conv_softmax/model.c ./work
    @cp ./tests/cases/conv_softmax/sbrk.c ./work
    @cp ./tests/cases/conv_softmax/test_data.h ./work
    @cp -r ./tests/cases/conv_softmax/stubs ./work/stubs
    @find ./tests/lib/ -maxdepth 1 -type f -exec cp {} ./work/ \;
    @cd ./work && make -s clean && make -s all \
        CPU_ARCH_FLAG_0=c906fd \
        ENDIAN_MODE=little-endian \
        CASENAME=conv_softmax \
        FILE=bare_main \
        EXTRA_CFLAGS="-DSHL_BUILD_RTOS -isystem stubs \
            -I$(CSI_NN2_INSTALL)/include \
            -I$(CSI_NN2_INSTALL)/include/csinn \
            -ffunction-sections -fdata-sections" \
        EXTRA_LDFLAGS="-Wl,--gc-sections -Wl,-z,muldefs \
            $(CSI_NN2_INSTALL)/lib/libshl_c906_rtos.a"
```

Key flags:
- `-Wl,--gc-sections` — Remove unused functions (critical for code size)
- `-Wl,-z,muldefs` — Allow our stubs to override library symbols

### 5.3 `smart_run/logical/tb/tb.v`

Increased `MAX_RUN_TIME` from 700ms to 30s to allow enough simulated time for
inference.

## 6. Step 4 — Build and Run

```bash
cd smart_run

# Set up environment
export PATH="<cmake-path>:<toolchain-path>/bin:$PATH"
export CODE_BASE_PATH=<repo>/C906_RTL_FACTORY
export TOOL_EXTENSION=<toolchain-path>/bin

# Compile RTL (once)
make compile SIM=vcs DUMP=off

# Build software
make buildcase CASE=conv_softmax

# Run simulation
cd work && ./simv +no_save
# Or use: make runcase CASE=conv_softmax SIM=vcs
```

**Expected output:**
```
  ********* Init Program *********
  ********* Wipe memory to 0 *********
  ********* Read program *********
  ********* Load program to memory *********

**********************************************
*    simulation finished successfully        *
**********************************************
```

**Result:** `work/run_case.report` should contain `TEST PASS`.

**Performance:** ~7.7 seconds wall time, ~42.6 ms simulated time.

## 7. Debugging Simulation Timeouts — The Hard Part

This section documents the most difficult aspect of the bring-up: diagnosing
and fixing simulation hangs that caused the testbench watchdog to fire.

### 7.1 The Symptom

After successfully passing session initialization and tensor setup, the
simulation would hang indefinitely when `csinn_conv2d()` was called.  The
testbench watchdog (which monitors retired instructions) would eventually fire
after detecting no progress, terminating the simulation with a timeout.

### 7.2 Root Cause — Indirect Jump (Tail Call) Hang

The root cause was traced to **indirect jump instructions (`jr`) used for
tail-call optimization** through function pointer tables in BSS memory.

Specifically, `shl_get_runtime_callback()` in the CSI-NN2 library looks up a
callback function from a table:

```c
// In csi-nn2/source/nn2/setup.c
void *shl_get_runtime_callback(struct csinn_session *sess, int op) {
    void *(*func)() = shl_runtime_callback_table[sess->base_api];
    return func(op);   // Compiled as: jr a5 (tail call)
}
```

At `-Os`, GCC optimizes this into a **tail call** — instead of `call` + `ret`,
it uses a single `jr` (jump-register) instruction.  The CPU loads the function
pointer from BSS memory into a register and jumps to it.

**This hangs the C906 RTL.**  The CPU enters a state where no instructions
retire, triggering the watchdog.

### 7.3 Diagnosis Methodology

Debugging was done via **incremental testing** — a `MINIMAL_TEST` macro
controlled how far into the CSI-NN2 initialization the code would proceed
before exiting:

| Level | What runs | Result |
|-------|-----------|--------|
| 4 | `csinn_session_init` only | ❌ HANG |
| 4 + override | Same, with `shl_get_runtime_callback` override | ✅ PASS |
| 5 | + tensor allocation | ✅ PASS |
| 6 | + `conv2d_init` + `softmax_init` | ✅ PASS |
| 66 | + `set_tensor_entry` | ✅ PASS |
| 67 | + `set_input` | ✅ PASS |
| 68 | + `csinn_conv2d` (graph building) | ❌ EXCEPTION |

The initial hang at level 4 was isolated to `shl_get_runtime_callback`.
A direct call to the same target function (`shl_gref_runtime_callback`)
worked, proving the function itself was fine — only the **indirect jump
through the table** was broken.

### 7.4 Fix 1 — Override `shl_get_runtime_callback` (Local)

A direct override was added in `sbrk.c` that calls the target function
directly instead of through the table:

```c
void *shl_get_runtime_callback(struct csinn_session *sess, int op) {
    if ((sess->base_run_mode == CSINN_RM_CPU_GRAPH &&
         sess->base_api == CSINN_REF) ||
        sess->base_run_mode == CSINN_RM_CPU_BASE_HYBRID) {
        return shl_gref_runtime_callback(op);  // Direct call, not via table
    }
    return NULL;
}
```

Linked with `-Wl,-z,muldefs` to allow overriding the library symbol.

**This fixed the session_init hang**, but the same class of bug reappeared
later at `csinn_conv2d` (level 68), where other indirect calls through
function pointers were also compiled as tail calls.

### 7.5 Fix 2 — `-fno-optimize-sibling-calls` (Global)

Overriding every function with indirect calls was not scalable.  The
**definitive fix** was to rebuild the entire CSI-NN2 library with:

```
-fno-optimize-sibling-calls
```

This GCC flag **disables tail-call optimization**, forcing all function
pointer calls to use the standard `call` + `ret` sequence instead of `jr`.

Before:
```asm
shl_get_runtime_callback:
    ld   a5, shl_runtime_callback_table(a1)
    jr   a5              # tail call — HANGS on C906 RTL
```

After:
```asm
shl_get_runtime_callback:
    ld   a5, shl_runtime_callback_table(a1)
    jalr ra, a5, 0       # regular call — WORKS
    ret
```

### 7.6 Fix 3 — Disable `SHL_DEBUG` and `SHL_TRACE`

After fixing the tail-call bug, there was still a linker error: the trace
function `shl_trace_move_events` was referenced directly (not through the
`SHL_TRACE_CALL` macro) in `graph_ref/setup.c`, but the implementation was
guarded by `#ifdef SHL_TRACE` in `shl_profiler.c`.

Rather than enabling `SHL_TRACE` (which pulls in file I/O code), both
`SHL_DEBUG` and `SHL_TRACE` were disabled entirely for the ELF build, and a
stub was provided for the lone unguarded call:

```c
// In sbrk.c
void shl_trace_move_events(struct shl_trace *from, struct shl_trace *to) {}
```

### 7.7 The crt0.s Vector Table Bug

An additional source of crashes was the startup code's exception vector table.
`crt0.s` populates the table with `.long` (4-byte) entries, but the exception
handler reads them with `ld` (8-byte load).  Any exception → garbage address
→ infinite exception loop.

**Fix:** Install a custom trap handler in `bare_main.c` that unconditionally
jumps to `__fail`:

```c
static void __attribute__((naked, aligned(4))) trap_handler(void) {
    __asm__ volatile(
        "la t0, __fail\n"
        "jr t0\n"
    );
}
// Installed via: csrw mtvec, trap_handler
```

### 7.8 Timeline of the Debugging Process

1. **Initial build** — Library compiles, test case links, simulation hangs
   immediately (watchdog timeout at ~53s wall time)
2. **Isolated to `csinn_session_init`** — Binary search with `MINIMAL_TEST`
   levels narrowed it to the first `shl_get_runtime_callback` call
3. **Identified tail-call pattern** — Disassembly showed `jr a5` instead of
   `jalr ra, a5, 0`; direct call to same function worked
4. **Local override** — Unblocked session init; graph building progressed
   further through tensor setup, conv2d_init, softmax_init
5. **Second hang at csinn_conv2d** — Same class of bug in other indirect calls
6. **Global fix** — Rebuilt library with `-fno-optimize-sibling-calls`
7. **Linker error** — Unguarded `shl_trace_move_events` call; fixed with stub
8. **Full inference passes** — `TEST PASS` in 7.7 seconds

## 8. Summary of Workarounds

| Problem | Root Cause | Fix |
|---------|-----------|-----|
| Simulation hang on indirect jumps | C906 RTL bug with `jr` tail calls through BSS function pointers | `-fno-optimize-sibling-calls` globally + local override of `shl_get_runtime_callback` |
| Can't compile RVV-optimized kernels | GCC 14.3 supports RVV 1.0 only; C906 has RVV 0.7.1 | Use `CSINN_REF` backend (reference C code only) |
| `printf`/file I/O in library | `SHL_DEBUG` and `SHL_TRACE` call `printf`/`vfprintf`/`fopen` | Disable both for bare-metal builds |
| Missing `sys/syscall.h` | `shl_profiler.h` includes it; not in newlib | Provide empty stub header |
| `shl_trace_move_events` linker error | Called directly, not via `SHL_TRACE_CALL` macro | Provide empty stub function |
| Exception causes infinite loop | `crt0.s` vector table word-size mismatch | Custom trap handler via `mtvec` CSR |
| Binary too large for testbench memory | Default build includes all ops + debug code | `-Os`, `-ffunction-sections`, `-Wl,--gc-sections` |
